📌 Project Overview

In this project, I designed and implemented an ETL data integration pipeline to support reporting and analysis. Data was collected from multiple structured sources (Orders, Customers, and Products), cleaned, transformed, and combined to produce a reliable, analysis-ready dataset using Microsoft Excel and Power Query.

🎯 Project Objectives

  • Extract data from multiple structured datasets into Power Query.
  • Clean and standardise values to improve consistency and usability.
  • Handle missing values and incomplete records appropriately.
  • Combine related datasets using key-based merge operations.
  • Create transformation logic such as custom calculations for reporting.
  • Apply data quality checks to validate accuracy and completeness.
  • Load a final clean and merged dataset for analysis and dashboarding.

🧰 Tools & Dataset

  • Tool: Microsoft Excel
  • Transformation Tool: Power Query
  • Datasets Used: Orders, Customers, Products
  • Project Theme: ETL for Data Analysts
  • Output File: Week7_ETL_Workbook_Merged.xlsx

🔁 ETL Pipeline Workflow

The project followed the standard ETL structure of Extract, Transform, and Load, tailored to a practical analyst workflow in Excel.

1

Extract

Imported the Orders, Customers, and Products datasets into Power Query as separate queries, preparing them for transformation and integration.

2

Transform

Cleaned missing values, adjusted formats, created a custom Total Sales calculation, and applied merge logic to combine related records across the datasets.

3

Load

Loaded the final validated merged table into Excel as a structured dataset ready for reporting, analysis, and business intelligence tasks.
